# 内容平台发布能力调研

更新时间：2026-07-18

## 结论

Notavia 可以分两条路径实现发布：

1. **官方接口直发**：优先接入抖音、哔哩哔哩。部署者需要申请平台应用，创作者通过平台授权；每次发布前仍展示标题、正文、媒体、目标账号和权限，并由用户点击最终确认。
2. **辅助发布**：知乎、小红书、微信视频号先提供格式检查、复制正文、下载媒体包、打开官方创作页和发布后登记。不要保存 Cookie、模拟扫码登录或在后台无感发布。

## 平台判断

| 平台 | 当前结论 | 依据与限制 |
| --- | --- | --- |
| 抖音 | 可通过官方 API 发布视频或图片 | [内容发布方案](https://open.douyin.com/platform/resource/docs/ability/content-management/douyin-publish-solution/)明确支持审核通过的第三方应用；`video.create` 需要用户 OAuth 授权，而且[创建视频接口](https://open.douyin.com/platform/resource/docs/openapi/video-management/douyin/create/create-video)要求每次代发布都让用户明确感知。 |
| 哔哩哔哩 | 可通过官方开放平台发布视频和专栏 | [开放平台](https://openhome.bilibili.com/doc)列出账号授权、视频稿件发布和专栏稿件发布。应用需完成身份/资质审核并关联 UP 主；具体准入以平台审核结果为准。 |
| 知乎 | 未确认有面向创作者的公开发布 API | 当前[开发者文档](https://developer.zhihu.com/docs?key=authorization)公开的是数据接口鉴权，未找到文章投稿接口。先做辅助发布。 |
| 小红书 | 未确认有通用创作者笔记发布 API | [小程序开放平台](https://miniapp.xiaohongshu.com/home)提到的是小程序场景中的笔记入口；服务商授权文档也围绕商家小程序，并非个人创作者投稿。先做辅助发布。 |
| 微信视频号 | 未找到公开的普通创作者投稿 API | 微信公开开发文档中未检索到视频号投稿接口。先做辅助发布，并持续关注官方开放能力。 |

## 推荐产品设计

- 增加“平台账号”页面，只存储加密后的 OAuth Token，不接触账号密码和 Cookie。
- 授权范围按平台、账号和能力展示，支持随时解除授权并删除 Token。
- 发布分为“准备 → 预览 → 明确确认 → 上传/发布 → 审核状态 → 数据回流”，禁止定时任务绕过确认。
- 官方接口不可用时，生成平台专用发布包：正文、封面、图片/视频、标签和核对清单；一键复制后打开官方创作中心。
- 自托管实例需要可公开访问的 HTTPS OAuth 回调地址。纯局域网部署可由部署者自备域名和平台应用；不应把所有用户导向项目维护者持有的共享密钥。

## 实施顺序

1. 先完成五个平台的辅助发布，统一预览和发布确认界面。
2. 申请抖音与哔哩哔哩开发者资质，验证 OAuth、沙盒和审核流程。
3. 接入发布状态与数据回流，再评估是否值得维护官方直发。
4. 每季度复查知乎、小红书和微信视频号官方文档，开放后再接入。
